# 🤝 Guía de Contribución

¡Gracias por tu interés en contribuir a EcclesiApp! Este documento proporciona guías y directrices para contribuir al proyecto.

## 📋 Código de Conducta

Este proyecto se adhiere a un código de conducta que esperamos que todos los contribuyentes cumplan.

## 🚀 Cómo Contribuir

### 1. Fork el Repositorio
```bash
git clone https://github.com/tu-usuario/hol.git
cd hol
```

### 2. Crear una Rama
```bash
git checkout -b feature/nombre-de-la-caracteristica
```

### 3. Hacer Cambios
- Sigue el estilo de código existente
- Escribe commits claros y descriptivos
- Agrega tests para nuevas características
- Actualiza la documentación según sea necesario

### 4. Enviar un Pull Request
- Describe los cambios de manera clara
- Referencia los issues relacionados
- Espera la revisión del equipo

## 📝 Estándares de Código

### TypeScript
- Usa tipos explícitos siempre que sea posible
- Evita `any`
- Sigue `strict` mode

### Commits
```
feat: descripción breve
fix: descripción breve
docs: descripción breve
style: descripción breve
refactor: descripción breve
test: descripción breve
chore: descripción breve
```

### Rama de Desarrollo
- `main` - Producción
- `develop` - Desarrollo
- `feature/*` - Nuevas características
- `fix/*` - Correcciones de errores
- `docs/*` - Documentación

## 🧪 Testing

```bash
# Backend
cd backend
npm run test

# Frontend
cd frontend
npm run test
```

## 📚 Documentación

Si agregas una nueva característica, actualiza:
- README.md
- Documentación relevante en `/docs`
- Comentarios de código donde sea necesario

## 🐛 Reportar Bugs

Crea un issue con:
- Descripción clara del problema
- Pasos para reproducir
- Comportamiento esperado
- Comportamiento actual
- Screenshots si es relevante

## 💡 Sugerir Características

Abre un issue con:
- Descripción de la característica
- Casos de uso
- Beneficios potenciales
- Ejemplos si es posible

## 📞 Preguntas

Si tienes preguntas, abre un issue o contacta con el equipo de desarrollo.
